Contexto geral

Pretende-se criar um site web interno para uma empresa de segurança eletrónica, que permita gerir clientes, equipamentos e avarias/intervenções, replicando e melhorando o funcionamento atual de um workspace interno já existente.

O site será usado diariamente pela equipa administrativa e técnica para registo, acompanhamento e fecho de trabalhos.

Objetivo do site

Criar uma plataforma simples e funcional para:

gerir clientes

consultar rapidamente informações por NIF

registar avarias e assistências

atribuir técnicos

acompanhar o estado dos trabalhos

manter histórico (datas, fotos e notas)

O foco é uso interno, não público.

Estrutura principal do site
1. Página principal – Gestão de Clientes e Avarias

Uma tabela/listagem central com os seguintes campos:

Dados do Cliente

Cliente (nome da empresa ou particular)

NIF

Morada

Telefone

E-mail

Serviços e Equipamentos

Serviço contratado (ex.: Videovigilância, Videoporteiro, Alarme, etc.)

Equipamento(s) (ex.: modelo do equipamento)

Número de série

Gestão de Avarias / Trabalhos

Estado (campo de status com fluxo definido)

Registo de assistência (data e hora)

Técnico responsável

Fotos da avaria (upload de imagens)

Notas / observações técnicas

Estados do trabalho (fluxo)

O sistema deve usar um campo de estado único, com estas opções:

A fazer – trabalho registado mas ainda não iniciado

Em curso – técnico já está a intervir

A faturar – trabalho concluído tecnicamente, pendente de faturação

Concluído – processo totalmente fechado

Assistência – intervenção adicional após um trabalho concluído

O estado deve ser facilmente filtrável.

Funcionalidades essenciais

Pesquisa rápida por cliente ou NIF

Filtros por estado, técnico e serviço

Possibilidade de:

criar novos clientes

registar novas avarias

atualizar estados

anexar fotos

Visual simples em formato de tabela

Preparado para crescer (ex.: login no futuro)

Regras de funcionamento importantes

O campo “Registo de assistência” é preenchido quando um trabalho entra em estado “Assistência”

Cada linha representa um registo ativo de cliente + equipamento + intervenção

O sistema deve privilegiar rapidez e clareza, não complexidade

Tom e estilo do site

Interface limpa e profissional

Pensado para uso diário em escritório ou telemóvel

Sem excesso de funcionalidades

Linguagem clara (português de Portugal)

NOTA IMPORTANTE (para o Replit / AI)

Este site deve espelhar um sistema interno já existente, não reinventar processos.
O objetivo é transformar uma gestão atual em workspace interno numa aplicação web dedicada, mantendo a simplicidade.
